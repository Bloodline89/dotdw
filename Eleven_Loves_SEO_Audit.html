Solution: Manual AggregateRating Schema

Add the following code to product.liquid in your Shopify theme:

{% if product.metafields.klaviyo.reviews_count > 0 %} <script type="application/ld+json"> { "@context": "https://schema.org", "@type": "Product", "name": "{{ product.title }}", "aggregateRating": { "@type": "AggregateRating", "ratingValue": "{{ product.metafields.klaviyo.reviews_average }}", "reviewCount": "{{ product.metafields.klaviyo.reviews_count }}", "bestRating": "5", "worstRating": "1" } } </script> {% endif %}

🔗 Product Recommendation URL Issue

Effort: Low Impact: High 🔴 Critical Priority Owner: Developer

Problem Identified

Shopify's built-in product recommendation engine appends tracking parameters to 76.5% of internal product links in "Related products" and "Pairs well with" sections.

https://elevenloves.co.uk/products/lizzy-rib-jersey-dress-chocolate?pr_prod_strat=pinned&pr_rec_id=710ef3822&pr_rec_pid=15189090894208

Impact Analysis

Impact Area Description Negative Effect
Link Equity Dilution PageRank passed inefficiently through parameter URLs 5-15% loss per link
Crawl Budget Waste Googlebot crawls thousands of unnecessary parameter URLs ~765 extra pages per 1,000 links
GSC Clutter "Alternate page with proper canonical tag" report filled with noise Obscures real issues
User Experience Long, unattractive URLs less shareable Reduced trust

Solution: Strip URL Parameters

Implementation Steps

  1. Backup Your Theme: Navigate to Online Store > Themes > Actions > Duplicate to create a backup before making changes
  2. Open Theme Code Editor: Go to Online Store > Themes > Edit Code
  3. Search for Product URL References: Use the search function to find instances of product.url or card_product.url within <a> tags
  4. Modify Each Instance: Apply the URL cleaning filter as shown below
  5. Test in Preview Mode: Navigate to product pages and hover over related product links to verify URLs are clean
  6. Publish Changes: Once verified, publish the updated theme

Files to modify: card-product.liquid, product-recommendations.liquid, product-item.liquid, featured-products.liquid

{{ card_product.url }} {{ card_product.url | split: '?' | first }}
⚠️ Important Note: Theme updates from Shopify will likely overwrite these custom changes. You will need to re-apply this fix after each theme update.

📚 The Five Principles of Semantic Internal Linking

1

Topical Relevance Over Link Volume

Quality connections, not quantity

The Principle: Every internal link should answer the question: "Would a user on this page genuinely benefit from visiting the linked page?"

Why It Matters:

  • Search engines use internal links to understand content relationships and topical authority
  • Links from topically related pages carry more weight than links from unrelated pages
  • Google's algorithms recognise semantic relationships between content

❌ Non-Semantic Links

  • Homepage → Random product
  • Footer → Every page equally
  • Sidebar → Unrelated promotions
  • Tulle collection → Gift cards

✅ Semantic Links

  • Collection → Related subcollection
  • Blog about tulle → Tulle collection
  • Product → Complementary products
  • Tulle collection → Styling guide
2

Hierarchical Structure (Hub & Spoke Model)

Organised topical clusters

The Principle: Organise content into topical clusters with clear parent-child relationships. Hub pages link down to spoke pages, and spoke pages link back up.

HUB PAGE
/collections/skirts
(Broad Topic)
SPOKE
/maxi-skirts
SPOKE
/tulle-skirts
SPOKE
/mini-skirts
SPOKE
/petite-skirts

How to Apply:

  1. Identify your hub pages (main category/collection pages)
  2. Map all related spoke pages (subcategories, product types, styles)
  3. Ensure bidirectional linking: Hub → Spokes AND Spokes → Hub
  4. Cross-link between related spoke pages
3

Contextual Anchor Text

Descriptive, keyword-relevant link text

The Principle: The clickable text of a link should describe the destination page's content using natural, keyword-relevant language.

Type Example When to Use
Exact Match "tulle maxi skirts" Sparingly, when natural
Partial Match "our collection of tulle styles" Primary approach
Branded "Eleven Loves tulle range" Brand reinforcement
Natural/Descriptive "explore flowing silhouettes for weddings" Content-rich contexts

❌ Poor Anchor Text

  • "Click here" - No semantic value
  • "Read more" - Generic
  • "This page" - Meaningless
  • "tulle tulle tulle" - Keyword stuffing

✅ Good Anchor Text

  • "Browse our recycled tulle collection"
  • "Discover styling ideas for weddings"
  • "Shop sustainable fashion"
  • "Explore elegant maxi skirt styles"
4

Link Equity Distribution

Strategic flow of ranking power

The Principle: Internal links pass "link equity" (ranking power) from one page to another. Strategic linking ensures your most important pages receive the most equity.

Homepage
Most Equity
↓
Collections
Collections
Collections
↓
Products
Products
Products
↓
Blog Posts
⚠️ Often Neglected

Current Issues for Eleven Loves:

  • 78 collection pages have only 2-3 internal links (starved of equity)
  • orphaned URLs (audit required) receive zero link equity
  • Blog posts at crawl depth 5-9 are isolated from main site
5

User Journey Alignment

Guide users from awareness to conversion

The Principle: Internal links should guide users through a logical journey that matches their intent, moving them from awareness to consideration to conversion.

Stage User Intent Link Strategy
Awareness "What tulle skirts are available?" Homepage → Category collections
Consideration "How do I style a tulle skirt?" Collection → Blog/styling guides
Comparison "Which colour suits me?" Product → Related products/variants
Decision "I want this one" Product → Add to cart
Post-Purchase "How do I care for it?" Confirmation → Care guide content
